# Feature Request: 支援多端數據庫同步功能 (Database Synchronization)

## 💡 問題與背景 (Background)
目前 **SayIt** 的所有分頁配置（`sayit-custom-tabs-v2`）與寫作內容（`sayit-tab-content-<tabId>`）均完全儲存於瀏覽器的 `LocalStorage` 中。

雖然這種設計極具隱私且速度極快，但在以下情境中會遇到限制：
1. **跨裝置使用**：無法在電腦編輯後，於手機上無縫繼續閱讀或寫作。
2. **資料遺失風險**：若瀏覽器快取被主動清除、過期，或在無痕模式下使用，將導致辛苦記錄的內容遺失。
3. **備份需求**：缺乏一鍵備份與還原的雲端機制。

因此，極需引進**數據庫同步 (Database Synchronization) 功能**，以提升應用的實用性與安全性。

---

## 🎯 功能目標 (Goals)
1. **多端同步**：支援多台裝置（Desktop、PWA 安裝版、手機瀏覽器）之間的內容無縫同步。
2. **隱私與去中心化**：秉持 SayIt 的 Serverless 與隱私優先精神，優先提供「Bring Your Own Storage (自備存儲)」選項。
3. **離線優先 (Offline-first)**：不影響現有的離線寫作體驗，僅在有網路且完成寫作時於背景靜默同步，並支援離線修改、上線後自動同步。

---

## 🛠️ 建議技術方案 (Proposed Solutions)
為了兼顧隱私、易用性與技術門檻，建議支援以下幾種同步媒介：

### 1. GitHub Gist 同步 (推薦，低成本且隱私友好)
*   **做法**：使用者於設定面板填入個人 GitHub Token，SayIt 自動在使用者帳號下建立一個隱藏的 Gist。
*   **優點**：完全免伺服器維護，資料歸使用者所有，免費且穩定。

### 2. 自訂 WebDAV 協定同步
*   **做法**：支援標準 WebDAV 協議，讓使用者連接到自己的 Nextcloud、堅果雲 (Jianguoyun) 或 WebDAV 相容雲端硬碟。
*   **優點**：適合注重隱私的極客群體。

### 3. Supabase / Firebase (BaaS) 同步
*   **做法**：使用者建立免費的 Supabase/Firebase 專案，將 API URL 與 Service Key 貼入 SayIt。
*   **優點**：支援多帳戶實時同步，甚至可延伸出協同編輯功能。

### 4. 自建輕量級同步伺服器 (Self-hosted)
*   **做法**：另建一個簡單的 Node.js + SQLite 後端專案，提供 Docker 部署，SayIt 可輸入該自建伺服器 URL 進行同步。

---

## 🔄 同步邏輯與衝突解決 (Sync Logic & Conflict Resolution)
為了確保多端修改時資料不被錯誤覆蓋，同步機制需具備：
1.  **時間戳記與版本控制**：
    *   在分頁和內容的資料結構中，新增 `updatedAt` 欄位。
    *   每次自動存檔時更新本地時間戳記。
2.  **衝突檢測 (Conflict Detection)**：
    *   同步前先拉取雲端狀態，比對雲端 `updatedAt` 與本地 `updatedAt`。
3.  **衝突解決 (Conflict Resolution)**：
    *   **無衝突**：若一端未變更，則以較新的一端為準（Last-Write-Wins）。
    *   **有衝突**：若兩端在上次同步後皆有更新，則彈出「衝突解決對話框」（可沿用 AI 結果比對的 Side-by-Side UI），顯示本地與雲端差異，讓使用者選擇「保留本地」、「覆蓋為雲端」或「手動合併」。
4.  **離線佇列 (Offline Queue)**：
    *   在網路斷開時，將待同步的變更寫入本地佇列，一旦監聽到網路恢復（`online` 事件），自動啟動背景同步。

## 🎨 UI/UX 設計構想 (UI/UX Design)
1.  **同步狀態列**：
    *   在標題列（或原 `save-state` 旁）新增一個精緻的小圖示 🔄。
    *   支援不同狀態：`雲端同步中...`、`已與雲端同步`、`同步失敗`、`檢測到衝突`。
2.  **同步設定面板**：
    *   整合至現有的設定彈窗（或新增獨立的「數據同步」分頁）。
    *   下拉選單選擇同步類型（None / GitHub Gist / WebDAV / Supabase / 自建伺服器）。
    *   相應的輸入欄位（Token, URL, 帳密），並提供「記住設定」勾選框（安全儲存於本地）。
    *   提供「立即同步」與「中斷連線」按鈕。
3.  **衝突處理介面**：
    *   採用毛玻璃風格的 Modal，左右分欄顯示本地與雲端衝突的文字內容，並用紅色/綠色高亮差異行。
